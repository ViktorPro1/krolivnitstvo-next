# Деплой проєкту

Проєкт «Кролівництво від А до Я» розгортається як сучасний Next.js-додаток, тому найкраще підходить Vercel або будь-який хостинг з підтримкою Node.js.

## Рекомендований варіант — Vercel

1. Підключіть репозиторій до Vercel.
2. Вкажіть стандартну збірку Next.js:
   - Build command: `npm run build`
   - Output directory: не потрібно вказувати вручну, Vercel автоматично визначає `.next`
3. Додайте змінні середовища:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`

## Локальний build

```bash
npm install
npm run build
npm run start
```

## Змінні середовища

У Vercel перейдіть у Settings → Environment Variables і додайте ті ж значення, що використовуються локально у `.env.local`.

## Рекомендації щодо публікації

- статичні файли з `public/` віддаються безпосередньо;
- `robots.txt`, `sitemap.xml`, `llms.txt` і `llms-full.txt` мають бути доступні з кореня сайту;
- після змін у SEO або метаданих бажано перевірити сторінку в браузері та в Search Console.

## Альтернативні варіанти

Якщо хостинг не є Vercel, важливо, щоб він підтримував:

- Node.js-сервер для Next.js;
- змінні середовища;
- роботу з `public/`-ресурсами.
