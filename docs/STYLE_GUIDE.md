# Style Guide

Цей документ описує правила оформлення коду для проекту.

## Іменування файлів

- Компоненти: `PascalCase`, наприклад `Header.tsx`.
- Звичайні модулі: `camelCase`, наприклад `usePublicPresence.ts`.
- CSS/SCSS: можна використовувати `kebab-case`, наприклад `AssistantPromo.css`.

## Компоненти

- Використовувати функціональні компоненти.
- Розділяти логіку та представлення.
- Уникайте занадто великих компонентів.
- Використовуйте `props` для передачі даних.

## TypeScript

- Надавайте типи для `props` та `state`.
- Використовуйте `interface` або `type` для складних структур.
- Уникайте `any`, якщо можна застосувати конкретний тип.
- Використовуйте `type` для об’єктних типів і `interface` для контрактів.

## Стилі

- Використовуйте модульні стилі, якщо це можливо.
- Уникайте глобальних селекторів, які можуть конфліктувати.
- Іменуйте CSS класи логічно та зрозуміло.

## ESLint

- Запускайте `npm run lint` перед створенням pull request.
- Виправляйте попередження та помилки.
- Дотримуйтесь конфігурації в `eslint.config.js`.
