# Архітектура проєкту

Цей документ описує, як побудовано проєкт «Кролівництво від А до Я» у його сучасній Next.js-версії.

## Основна ідея

Проєкт поєднує:

- контентну платформу про кролівництво;
- SEO-орієнтовану структуру сторінок;
- сучасний UI на React і Next.js;
- підключення Supabase для авторизації та даних.

## Структура `src`

- `app/` — основна частина проєкту на App Router.
  - `app/layout.tsx` — базовий layout.
  - `app/page.tsx` — головна сторінка.
  - `app/_pages/` — окремі сторінки контенту та функціональні розділи.
  - `app/components/` — UI-компоненти, які використовуються по всьому сайту.
- `components/` — загальні компоненти, які не прив’язані до app-router-структури.
- `data/` — статичні дані, контентні масиви, секції, метадані.
- `hooks/` — кастомні React hooks.
- `lib/` — допоміжні модулі, конфігурація Supabase, утиліти.
- `context/` — глобальний контекст для стану застосунку.

## Маршрути

Маршрути базуються на файловій структурі App Router:

- сторінки розміщуються у `src/app/` і підпапках;
- Next.js самостійно формує маршрути за файловою системою;
- для SEO використовуються метадані на рівні сторінок.

## Основні компоненти

- `Header` — навігація та логотип.
- `Footer` — нижній колонтитул.
- `Assistant` — допоміжний інтерфейс для користувача.
- `CookieConsent` — банер з погодженням на cookies.
- `WelcomePopup` — вітальне вікно.
- `UpdatePrompt` — повідомлення про оновлення.
- `Breadcrumbs` — навігаційні хлібні крихти.

## Auth / Supabase

- Авторизація реалізована через `src/app/lib/supabase.ts`.
- У проєкті використовуються змінні середовища з префіксом `NEXT_PUBLIC_`.
- На сторінках, де потрібен доступ до особистих даних, перевіряється сесія користувача.

## SEO та продуктивність

- проєкт використовує метадані через Next.js;
- є підтримка Open Graph і структурованих даних;
- сторінки орієнтовані на швидке завантаження та індексацію.

## Загальні принципи

- зберігати просту, зрозумілу структуру;
- ділити код на логічні модулі;
- підтримувати контент і UI в одному єдиному підході;
- не ускладнювати архітектуру без потреби.
