# Налаштування середовища

Цей документ описує, як правильно налаштувати середовище для розробки та деплою проєкту «Кролівництво від А до Я».

## Локальна розробка

### Встановлення

```bash
npm install
```

### Запуск сервера

```bash
npm run dev
```

За замовчуванням застосунок відкриється на:

```text
http://localhost:3000
```

## Змінні середовища

Проєкт використовує Supabase для авторизації та роботи з даними. У локальному середовищі додай файл `.env.local` з такими змінними:

```env
NEXT_PUBLIC_SUPABASE_URL=https://your-supabase-project-url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-supabase-anon-key
```

### Пояснення

- `NEXT_PUBLIC_SUPABASE_URL` — URL твого Supabase-проєкту.
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` — публічний анонімний ключ для клієнта.

> Зберігай секрети поза репозиторієм. Не додавай `.env*` до Git.

## Деплой

На Vercel або іншому хостингу додай ті самі змінні середовища у налаштуваннях проєкту.

## Важливі файли

- `package.json` — скрипти та залежності
- `next.config.ts` — конфігурація Next.js
- `tsconfig.json` — налаштування TypeScript
- `eslint.config.mjs` — правила ESLint

## Безпека

- не зберігай приватні ключі в Git;
- використовуйте `NEXT_PUBLIC_` лише для змінних, доступних клієнту;
- для чутливих даних використовуй серверні змінні, якщо це буде потрібно в майбутньому.
