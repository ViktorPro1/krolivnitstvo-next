# Testing

Цей документ описує тестову стратегію для проекту.

## План тестування

Навіть якщо поки немає тестів, рекомендується включити:

- unit tests для компонентів
- integration tests для ключових сторінок
- end-to-end tests для основного користувацького потоку

## Інструменти

- `Vitest` — для unit та integration тестів.
- `React Testing Library` — для тестування компонентів React.
- `playwright` або `cypress` — для e2e тестів (опціонально).

## Як запускати тестування

Після встановлення пакетів можна додати команду в `package.json`:

```json
{
  "scripts": {
    "test": "vitest",
    "test:watch": "vitest --watch"
  }
}
```

Запуск тестів:

```bash
npm run test
```

## Рекомендації

- Покривайте компонент `AppRoutes` та сторінки з авторизацією.
- Тестуйте логіку auth та випадки, коли користувач не авторизований.
- Перевіряйте рендеринг важливих UI-компонентів, таких як `Header`, `Footer`, `CookieConsent`.
